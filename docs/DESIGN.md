# CHIP-8 TETRIS 実装計画

chip8-lang (自作関数型言語) で TETRIS を実装する。

## ゲーム仕様

### 画面レイアウト (64×32 ピクセル)

```
┌────────────────────────────────────────────────────────────────┐
│                             64px                               │
│  ┌──────────────────────┐  ┌───────────────────────────────┐  │
│  │      PLAYFIELD       │  │          INFO AREA            │  │
│  │    20px × 30px       │  │                               │  │
│  │  (10列 × 15行)      │  │  SCORE: 0042                 │  │
│  │  1セル = 2×2px       │  │                               │  │
│  │                      │  │  NEXT:                        │  │ 32px
│  │                      │  │   ██                          │  │
│  │                      │  │  ██                           │  │
│  │                      │  │                               │  │
│  │                      │  │                               │  │
│  └──────────────────────┘  └───────────────────────────────┘  │
└────────────────────────────────────────────────────────────────┘
```

- **プレイフィールド**: 左側 20×30px (10列 × 15行、各セル 2×2px)
- **情報エリア**: 右側にスコアとネクストピース
- 1行目は壁の上端用に空ける

### テトロミノ (7種)

各ピースは 8×2px のスプライト (sprite(2)) で表現。
セルが 2×2px なので、4セル幅 × 1セル高のスプライトを使い、
回転は別スプライトとして定義する。

```
I: ████████    O: ████      T:  ██       S:  ████    Z: ████
                  ████       ██████         ████        ████

L: ██          J:     ██
   ██████        ██████
```

各ピースは最大4回転状態 = 最大 28 スプライト。
ただし O は1種、I と S と Z は2種なので、計 19 スプライト。

### 操作 (CHIP-8 キーパッド)

| キー  | QWERTY | 操作     |
|-------|--------|----------|
| 4 (Q) | Q      | 左移動   |
| 6 (R) | R      | 右移動   |
| 5 (W) | W      | 回転     |
| 8 (S) | S      | 即落下   |

### ゲームロジック

1. ランダムにピースを選択
2. 画面上部にピースを配置
3. 一定時間ごとに1行落下 (ディレイタイマー使用)
4. キー入力で左右移動・回転
5. 着地判定: 下に移動できない場合は固定
6. ライン消去: 行が埋まったら消去してスコア加算
7. ゲームオーバー: ピースが配置できない場合

### コア設計: 画面をゲーム状態として使用

CHIP-8 の `DRW` 命令は XOR 描画であり、衝突検出 (VF) を返す。
これを活用して、**画面自体をボード状態**として扱う:

- **ピース移動**: 現在位置で XOR 描画 (消去) → 新位置で XOR 描画 (再描画)
- **衝突検出**: 新位置に描画して VF をチェック。衝突なら再度 XOR で元に戻す
- **ライン消去**: 行全体を走査して塗りつぶし判定

この方式により、配列によるボード管理が不要になり、
chip8-lang の現在の制約 (配列が読み取り専用) 内で実装できる。

## chip8-lang の制約と対策

### 判明している制約

| 制約 | 影響 | 対策 |
|------|------|------|
| 関数呼び出し時にレジスタが破壊される | 呼び出し前後でローカル変数が消える | → chip8-lang に issue 登録。当面は関数を最小限に |
| 配列が読み取り専用 | ボード状態を配列で管理できない | → 画面を状態として使う方式で回避 |
| 乗除算が未実装 | 座標計算に掛け算が使えない | → 加算の繰り返しか、ルックアップテーブルで代替 |
| ローカル変数が最大15個 | 関数内で使える変数数に制限 | → 関数を分割して変数を節約 |
| `FX65` のセマンティクスが誤り | V0〜Vx まで全部読み込まれる | → chip8-lang に issue 登録 |

### chip8-lang に報告予定の issue

1. **関数呼び出し時のレジスタ退避/復帰がない** (critical)
2. **FX65 の誤用: 単一レジスタ読み込みに使っているが V0〜Vx が上書きされる** (critical)
3. **乗除算 (Mul/Div/Mod) が未実装** (enhancement)
4. **ミュータブルなグローバル変数がない** (enhancement)
5. **配列への書き込みができない** (enhancement)
